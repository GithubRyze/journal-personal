10.30 的日志中有记录，找到工作 11.3 上班。但在上班之前的 10.31 有公司让我去面试，我本来不打算去的，因为要我面试的公司名字高大上（香港城市大学研究院），我感觉面上的希望也不大并且下午我准备看 LOL S15 比赛。但是老婆在这公司附近上班，她想让我去试试，所以我就去试试了。

## 面试回顾
找到地方，进入办公室，空间还挺大，3楼人不多。在我之前就有人去面试了，所以我在外面应该约等了 20 分钟。坐好，面试官说约 20 分钟，首先自我介绍，然后问一些技术上的问题。问题现在回顾大约如下。
- git 冲突处理：这里是查看是否用过，并且是否使用规范
- 订单系统：考察业务和 mq 的使用，主要是 id, 幂等，对账，人工，监控，告警，消费失败重置，ACK 等知识。这个地方我表现应该很好，因为有研究过，并且我拿出了纸把流程和关键的地方在纸上画出来和写出来了。
- redis 分布式锁， set nx px 命令和使用 lua 脚本（开始回答了命令，后面他问还有其他方式没，换其他时候我可能想不到，那天感觉面试状态进入了就想到了 lua 脚本实现）
- dubbo 在数据块大要怎么处理，支持哪些协议。当时我就回复了 grpc，数据块大后面才晓得应该 protocolbuf （grpc），数据压缩，http2 io 复用
- http 调用的库， feign，okhttp，httpclient 这个应该是看是否真的写过代码，或者了解一些库。
- sse 相关，这个东西我忘记具体是什么，但是我之前工作中好像有印象可以实现后端更新推送到前端（websocket 也可以），我就回答了上面这些内容。

快结束了就问我有什么想问的，当时我一听就觉得应该是没戏了（因为在我看来这是要结束面试的流程，并且没有问期望待遇，上班时间），我就挑了一个问题然后当时也没怎么认真仔细的听（心里想的是结束吧，我要看 LOL S15 的比赛了）。最后没想到居然对我说对我的整体印象很不错，然后问了待遇和上班时间。最后我走了，我感觉没多少机会也就没怎么放心上。我看他手中的简历应该有 10 来份左右吧。

## 11.03
我入职了深圳乐福点科技有限公司，一个做针对老年人护理终端的企业。是一个外企，10 来个人，2 个后端开发，2 个硬件开发，三个 pm，1 个 hr，外加一个 老板。 当日就给我介绍了业务，给了些后端资料，让我自己看。

然后晚上我就打开招聘的软件看看 10.31 面试是否有进一步消息，没想到早上 11 点左右 hr 居然给我留言，要流水和毕业证。我就折腾了下就给了，另外还要个税记录，我想我上次可是 25，给你们只开了 19，另外我目前也有 18，你们给不给 offer 我也不是特别在意。

## 11.04
上班第二天，今天 cto 来了，我们一起开了一个会议。他们拿出一个 api 列表，然后针对这个上面内容进行分工，我是头一次看到这样的（因为 api 只是功能中的一部分）。后面有给我分配了 notification，support, faq 等模块吧。在会议中有讨论第三方登录的功能，这个我没做过但是我了解下，老板问我你做过没，我就说我了解一些，老板回复说：你做过就做过，没做过就没做过，了解些就是没做过咯。我战战兢兢的说：没做过。后面又到 notification 功能，说真我当时确实不清楚这个是具体做什么，脑子中只晓得通知（推送，email，feishu，钉钉）这些通知。然后老板又来一句：你们都没有用过 app 的 notificaiton 吗，就是这个这个，现在的年轻人都不用的，还做服务端开发。我只能是 哎！！！

会后我去找资料才发现原来的刚才的 notification，就是一般 app 中的通知管理中心（通过他们给的 api 列表）。我顿时无语了，他们自己都讲不清楚，也不了解做啥，怎么做。

当天下午 3 点，我又和研究员的 hr 通了一次电话。但是这个 hr 不太熟悉这些人事这些事情。晚上我又和上次面试的通了一次电话，细节不说了，因为今天下午的事情我有点不想在当前公司继续做了。

## 11.05
当天上午我继续工作，把模块中的数据表结构和逻辑梳理清楚了，并且开发了一些 api，利用 swagger 把接口文档化了。在这 3 天中我发现公司有以下这些问题：
- 老板对我的专业和能力不太信任，这是一种直觉没办法证明
- 同事设计了一套微服务的架构，但是我询问为何要这样做他有点说不上来，就只会说：外面都是这样做，你们之前不是这样吗。我认为这个架构太复杂了，并且只有 2 个人维护的情况下，并且业务可以说是非常简单了。
- 功能分析缺失，只会看 api 列表，都没梳理清楚功能闭环。比如：那里可以看到当前有多少用户，设备信息，用户和设备绑定关系，服务端从那里触发通知功能等等。
- 另外因为同事已经设计了一套架构，我不能说服他们改（这样就直接否认了同事的工作），另外我提了比如：老板从那里看设备信息，用户信息，下发通知这些。我同事给我一个数据表结构。难道到时候用 sql 给老板查出来。

通过以上这些地方，我感觉我的同事没有很成熟的架构思想以及系统功能分析的能力。只会无脑的堆砌技术，缺乏业务理解和技术理解，因此我非常担心项目后期问题多，老板肯定会让我帮忙一起。另外一个问题是老板不信任我。最后一个问题我们虽然可以重新设计架构和业务处理一些问题，但是实际执行可行性不大。
- 同事先设计好，推翻相当于打他的脸，后面极有可能不太好处
- 我没办法证明同事的架构有问题，同事可以说这是为了以后 100 w 数据量的架构
- 老板对我信任不足

当天下午 5 点左右，研究员给我发了 offer，我当时还在思考是否需要接受，因为这边的问题我已经看到了，同事也基本了解了（氛围还很不错）。研究院那边很多不确定性（具体的业务，稳定性，氛围这些），但是我对面试官的印象比较好。

当天我思考了很多方面，我最终决定接受研究院，因此晚上我就给 hr 发了微信，说自己能力不足不能胜任工作离职。 老板给我打了电话询问原因，

## 11.06
上午老板不在，下午老板来了，叫我到房间里面谈了下，我还是一样说自己能力不足。然后叫我同事过来简单说下，我拿着入职的资料出来了。然后收拾东西离开公司。

再后面同事问我为什么离职，我说能力不够，另外了解到他的工资居然比我高 2 k，我惊讶了，并且他还告诉我部署什么他都不懂，都说让朋友来搞的。到目前我还是很放松的，觉得离开也挺好。反正明天要入职新公司

## 11.07
新公司入职，小公司流程不规范也不复杂。人都没有认识，也没有人安排介绍之类的。就是入职有点尴尬。后面聊天发现他们都是 00 年左右的人，一个是 99 年后的，我感觉他们好年轻。
- 团队年轻，我是比较大的
- 领导基本不来公司，开始我还想怎么抱大腿
- 上班氛围轻松，时间 9，14，18。基本到点就走了
- 头天任务，编写一个小红书自动评论的工具。这个我没做过，但是之前公司中有接触，利用 ui 自动化框架去做

至此，新公司团队也要大，但是很多不规范，部署还是比较原始的 jar 包。这些都不是太问题，技术可以升级的。整体让我最满意的是上班时间，非常好。主要是不清楚能呆多久，会不会也裁员。毕竟现在经济环境非常差
